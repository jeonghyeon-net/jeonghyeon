토요일 저녁이 됐다. 기본 기능은 다 됐다. Jira 연동, 터미널, worktree 관리. 다 작동한다. 예상보다 빨리 됐다. 저녁에 친구들이랑 술 약속이 있어서 나갔다. 맥주를 마시면서 친구들한테 자랑했다. "나 오늘 앱 만들었어." "뭔 앱?" "Jira랑 터미널이랑 git worktree 통합한 앱. AI로 코딩하기 편하게." "오 대단하네. 근데 그게 왜 필요해?" 설명하려다가 귀찮아서 그냥 "나한테 필요해"라고 했다. 개발자 아닌 친구들한테 설명하는 건 더 귀찮다.

일요일. 숙취 해소하고 오후에 다시 앉았다. 마무리 작업을 했다. "다중 계정 지원해줘. 회사 Jira랑 개인 Jira 둘 다 연결할 수 있게." 엔터. 계정 관리 UI가 생겼다. "다크 테마 적용해줘. CSS 변수로 테마 시스템 만들어줘." 엔터. 다크 테마가 적용됐다. 눈이 편하다. "GitHub PR이랑 연동해줘. 이슈에서 바로 PR 열 수 있게." 엔터. GitHub API 연동이 추가됐다. 아, 이건 아직 버그가 좀 있다. 나중에 고치자.

앱이 완성됐다. 일주일도 안 걸렸다. 솔직히 실제로 코딩한 시간은 하루 정도인 것 같다. 나머지는 Claude Code가 했다. 나는 지시하고, 확인하고, 버그 있으면 고치라고 하고. 그게 끝이다. 예전 같으면 Jira API 문서 읽느라 반나절, xterm.js 예제 찾느라 반나절, PTY 연동에서 삽질하느라 이틀, 거기다 버그 잡느라 삼일. 최소 일주일은 빡세게 작업했을 거다. 근데 이번엔 그냥 날먹했다. AI가 다 해줬다.

이제 내 워크플로우는 이렇다. 아침에 출근해서 Jeonghyeon을 켠다. 하나의 앱. 왼쪽에서 오늘 할 티켓을 클릭한다. 바로 뜬다. 로딩 없다. Jira 웹에서 5초 걸리던 게 여기선 0.3초다. 티켓 내용을 읽는다. "Worktree 생성" 버튼을 누른다. 자동으로 worktree가 생기고, 터미널이 그 경로에서 열린다. 터미널에 claude를 친다. Claude Code가 뜬다. 티켓 내용을 복사할 필요 없다. 바로 옆에 보이니까. "이 기능 만들어줘"라고 치면서 옆의 티켓 내용을 참고한다. 눈만 움직이면 된다. 창 전환 없다. Command + Tab 안 눌러도 된다.

다른 티켓 보고 싶으면 옆에서 클릭하면 된다. 브라우저 탭 스무 개 뒤질 필요 없다. 터미널을 분할해서 동시에 여러 AI 에이전트를 돌릴 수 있다. 왼쪽에서 Claude Code, 오른쪽에서 테스트 돌리기. 작업이 끝나면 바로 티켓 상태를 "완료"로 바꾼다. Jira 새로고침 안 해도 된다. 흐름이 안 끊긴다. 편하다. 진짜 편하다.

이름은 그냥 내 이름 붙였다. Jeonghyeon. 정현. 프로젝트 이름 짓는 거 진짜 귀찮다. 뭔가 그럴듯한 이름 지어봤자 검색하면 다른 거 나온다. "DevFlow"? 이미 있다. "GitBuddy"? 이미 있다. "CodeStation"? 촌스럽다. 그냥 내 이름 붙이자. Jeonghyeon. 검색하면 나만 나온다. 내 앱이니까 내 이름 붙이는 거, 뭐가 어때서. 어차피 나 쓰려고 만든 거다.

이게 이 앱의 탄생 스토리다. 화려한 스타트업 창업기도 아니고, 새벽까지 코딩하며 꿈을 키운 이야기도 아니다. 그냥 귀찮아서 만들었다. 창 왔다 갔다 하는 거 귀찮고, worktree 어디 있는지 기억하는 거 귀찮고, Jira 로딩 기다리는 거 귀찮아서. AI가 코드 짜주는 시대에 맞는 도구가 없어서, 그냥 AI 시켜서 만들었다. 일주일도 안 걸렸고, 실제로 코딩한 건 하루 정도다. 좋은 세상이다. AI 만세.

혹시 너도 비슷한 워크플로우라면 한번 써봐. Claude Code나 Codex나 Gemini CLI 같은 AI 에이전트로 코딩하고, git worktree 써서 여러 작업 동시에 진행하고, Jira로 티켓 관리하고. 이 셋을 왔다 갔다 하느라 지쳤다면, Command + Tab을 하루에 백 번씩 누르면서 "내가 뭘 하고 있었지?" 생각하느라 멍해지는 경험을 했다면, 한번 써봐. 마음에 들면 쓰고, 마음에 안 들면 말고. 강요는 안 한다. 근데 나는 이거 만들고 나서 진짜 편해졌다. 출근해서 앱 하나만 켜면 되니까. 브라우저 안 켜도 되고, Jira 로딩 안 기다려도 되고, 터미널 창 열다섯 개 띄워놓고 어느 게 어느 건지 헤매지 않아도 된다. 그냥 한 화면에서 다 된다. 이게 얼마나 편한지는 써봐야 안다.

실행하려면 몇 가지가 필요하다. Node.js 22버전 이상, pnpm, 그리고 Rust. Node는 nvm 쓰면 편하다. 프로젝트 루트에 .nvmrc 파일 있으니까 nvm use만 치면 알아서 맞는 버전으로 바뀐다. pnpm은 npm install -g pnpm으로 설치하면 된다. Rust는 rustup.rs 가서 설치하면 된다. 다 설치했으면 터미널 열고 프로젝트 폴더로 가서 nvm use 치고, pnpm install 치고, pnpm dev 치면 앱이 뜬다. 빌드해서 설치 파일 만들고 싶으면 pnpm tauri build 치면 된다. macOS면 .dmg 파일이 나오고, Windows면 .exe가 나온다. 처음 실행하면 Jira 연결 설정하라고 나온다. 연결 이름 아무거나 넣고, Jira Base URL 넣고 (예: https://yourcompany.atlassian.net), 이메일 넣고, API 토큰 넣으면 된다. API 토큰은 Atlassian 계정 설정에서 발급받을 수 있다. id.atlassian.com 가서 Security 탭 들어가면 API tokens 메뉴 있다. 거기서 만들면 된다.

버그 있으면 GitHub Issues에 올려줘. Jira 말고 GitHub으로. Jira에 올리면 내가 Jira 로딩 기다려야 하잖아. 아이러니하게도 Jira 싫어서 만든 앱인데 Jira로 버그 관리하면 웃기다. GitHub Issues로 부탁한다. 버그 제보해주면 진심으로 고맙다. 근데 솔직히 말하면 나도 버그 고치는 건 Claude한테 시킬 거다. "이 버그 고쳐줘" 하고 이슈 링크 던져주고 엔터 치면 끝이니까. 그래도 버그를 발견해서 알려주는 건 사람이 해야 한다. AI는 버그를 만들 수 있지만, 사용자가 겪는 버그를 발견하는 건 사용자밖에 못 한다. 그게 당신이다. 부탁한다. 재현 방법 자세히 적어주면 더 고맙다. 스크린샷이나 에러 로그 있으면 더더욱 고맙다.

로고가 없다. 지금 앱 아이콘은 Tauri 기본 아이콘이다. 저 파란색 네모에 뭔가 들어있는 아이콘. 촌스럽다. 내 앱인데 기본 아이콘 달고 있으면 좀 그렇다. 누가 로고 좀 만들어주면 좋겠다. 디자인 센스 있는 분, 그래픽 작업 할 줄 아는 분, GitHub Issues나 이메일이나 뭐 어디로든 연락 주시면 감사하겠습니다. 보상은... 솔직히 돈은 없다. 대신 README에 크레딧 넣어드리고, 앱 About 화면에도 넣어드린다. 그리고 내가 진심으로 고마워할 것이다. 나는 코드는 짜는데 디자인은 정말 못 한다. 초등학교 때 미술 성적 나빴다. AI한테 로고 만들어달라고 시켜봤는데 AI도 로고는 좀 별로더라. 뭔가 기업 로고 같은 게 나온다. 개성이 없다. 이건 사람이 필요하다.

아이디어 있으면 던져줘. "이런 기능 있으면 좋겠다" 싶은 거 있으면 Issues에 올려줘. 예를 들어 "Linear 연동도 해줘" 같은 거. "Notion이랑 연동해줘" 같은 거. "터미널 테마 커스터마이징 되게 해줘" 같은 거. 뭐든 좋다. 구현은 내가 한다. 아니, Claude가 한다. 어쨌든 된다. 좋은 아이디어는 언제나 환영이다. 직접 기여하고 싶으면 PR 올려도 좋다. 코드 스타일 같은 건 크게 신경 안 쓴다. ESLint 돌려서 에러 안 나면 된다. 타입 에러 없으면 된다. 돌아가면 된다. 깔끔하면 더 좋고, 안 깔끔해도 리뷰하면서 같이 고치면 된다. 오픈소스는 함께 만들어가는 거니까. 혼자 만들면 외롭다. 같이 만들면 덜 외롭다. 기여해주시면 고맙다.
